# 自进化体系闭环 SOP

> 编码：KB_SOP_008_v1.0.0 | 版本：1.0.0 | 2026-06-27 | 状态：active
> 监管要求：IATF16949:2016 §7.5 成文信息控制
> 适用范围：OpenClaw 生态的自进化机制设计、运行、优化

## 1. 目的

定义 OpenClaw 自进化系统的闭环运行机制，确保系统能够：
- 通过「观察→决策→执行→评估→调整」的反馈循环实现能力增长
- Dreaming System 定期将 Session 数据转化为洞察
- MyBrain 沉淀永久积累，实现跨时间维度的能力提升
- 每个进化循环有记录、可追溯、可验证

## 2. 自进化三层架构

| 层级 | 组件 | 职责 | 周期 |
|------|------|------|------|
| 感知层 | Session / Heartbeat | 观察当下、发现问题 | 实时/30min |
| 加工层 | Dreaming System | 深度分析、提炼洞察 | 每日夜间 |
| 积累层 | MyBrain | 永久沉淀、跨时间复用 | 按需/每日 |

## 3. 闭环运行流程

```
┌─────────────────────────────────────────────────────────┐
│  感知层：Session / Heartbeat                            │
│  观察 → 记录 → 问题识别                                 │
└────────────────────┬────────────────────────────────────┘
                     │ 每日夜间汇聚
                     ▼
┌─────────────────────────────────────────────────────────┐
│  加工层：Dreaming System                               │
│  记忆整合 → 洞察提炼 → 归档建议                         │
└────────────────────┬────────────────────────────────────┘
                     │ 知识沉淀申请
                     ▼
┌─────────────────────────────────────────────────────────┐
│  积累层：MyBrain                                       │
│  评估 → 入库 → 卡片生成 → 受控文件登记                  │
└────────────────────┬────────────────────────────────────┘
                     │ 新卡片/规则
                     ▼
┌─────────────────────────────────────────────────────────┐
│  执行层：Agent / Cron                                  │
│  应用新规则 → 运行 → 反馈收集                           │
└────────────────────┬────────────────────────────────────┘
                     │ 新观察
                     ▼
              (回到感知层)
```

## 4. 各层操作规范

### 4.1 感知层（Session + Heartbeat）

**观察内容：**
- 任务完成情况（成功/失败/耗时）
- 错误模式（重复出现的错误类型）
- 效率数据（Token 消耗、响应延迟）
- 用户反馈（需求变更、满意度）

**记录规范：**
- 每次任务结束 → 写入 `daily/YYYY-MM-DD.md` 执行记录区
- 发现模式异常 → 写入 `inbox/问题单_YYYY-MM-DD.md`
- 记录格式：时间 | 任务 | 结果 | 关键洞察

**判定标准：**
- 单日内同一错误出现 2 次 → 触发进化需求
- 任务耗时环比增长 >30% → 触发效率审查
- 用户明确负面反馈 → 触发紧急评估

### 4.2 加工层（Dreaming System）

**输入：**
- `memory/sessions/` 中的 Session 记录
- `memory/daily/` 中的每日执行记录
- `inbox/` 中的待处理问题单

**处理流程：**
1. 整合过去 24 小时的所有记忆
2. 识别高频模式（成功路径 + 失败路径）
3. 提炼可复用的方法论（形成新 SOP 草稿）
4. 识别系统性问题（需要结构性改变）
5. 生成洞察报告（`memory/dreams/`）

**输出：**
- 洞察报告 → `memory/dreams/deep/YYYY-MM-DD.md`
- 建议入库的知识 → 触发知识入库流程
- 系统性问题 → 触发告警 SOP

**判定标准：**
- 洞察报告须含「核心发现」和「建议行动」
- 每次 Dreaming 须产出至少 1 条可执行建议
- 建议分两类：立即行动（24h）/ 计划行动（周度）

### 4.3 积累层（MyBrain）

**入库标准：**
- 洞察经过 2 次以上验证（不是偶发事件）
- 具备通用性（不只适用于当前单一场景）
- 符合卡片格式要求（摘要/详情/核心要点/相关）

**卡片生成：**
- 按 `KB_SOP_006`（知识入库 SOP）执行
- 编码：`KB_CAR_{类型}_{序号}_v1.0.0`
- 状态：从 `draft` 开始，评审后改为 `active`

**受控登记：**
- 新 active 卡片 → 更新 `system/受控文件清单.md`
- 新 active SOP → 更新 `system/SOP索引.md`

### 4.4 执行层（Agent + Cron）

**反馈收集：**
- 新卡片/SOP 应用后，持续收集运行数据
- 收集周期：新规则应用后 7 天内每日检查
- 收集内容：规则是否有效、是否需要调整

**调整触发：**
- 规则应用后 7 天，评估有效性
- 有效 → 保持，进入常规运行
- 部分有效 → minor 修订（KB_SOP_003）
- 无效 → major 修订或回滚

## 5. 进化速率控制

| 进化类型 | 说明 | 建议周期 |
|---------|------|---------|
| 微调 | 错别字、格式、描述优化 | 随时（patch） |
| 增量 | 新增小流程、补充案例 | 每周（minor） |
| 重构 | 结构性改变、新机制引入 | 每月评估（major） |

**约束：** 单一 Agent 的进化更新不超过每周 1 次（major），
避免频繁变更导致系统不稳定。

## 6. 进化效果评估

| 维度 | 指标 | 测量方式 |
|------|------|---------|
| 效率 | Token 消耗环比 | Cron 输出数据 |
| 质量 | 错误率环比 | daily/ 错误记录统计 |
| 知识积累 | 卡片新增数量 | 受控文件清单 |
| 自进化 | 规则自我改进数量 | 变更记录统计 |

**评估频率：** 每月末，CEO 读取 `system/受控文件清单.md` 和 `daily/` 数据，
生成「自进化月报」存入 `daily/月报_YYYY-MM.md`。

## 7. 禁止事项

- **禁止**在 Dreaming 未完成的情况下跳过洞察提炼直接入库
- **禁止**一次性进行多项 major 进化（分开评估）
- **禁止**将未验证的洞察当作已验证方法论使用（除非标注为试验性）
- **禁止**进化过程中删除历史记录（只增不减）

## 8. 记录要求

| 记录 | 保存位置 | 保存期限 |
|------|---------|---------|
| 每日执行记录 | `daily/YYYY-MM-DD.md` | 永久 |
| Dreaming 洞察报告 | `memory/dreams/deep/` | 永久 |
| 自进化月报 | `daily/月报_YYYY-MM.md` | 永久 |
| 变更记录 | 各文件底部变更记录区 | 永久 |
