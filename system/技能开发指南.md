# 技能开发指南

> 编码：KB_WI_006_v1.0.0 | 版本：1.0.0 | 2026-06-27 | 状态：active
> 适用范围：所有 OpenClaw Skill 的开发、评审、上线
> 负责人：CEO

## 1. 目的

规范 OpenClaw Skill 的开发流程，确保每个 Skill：
- 有清晰的功能边界和使用条件
- 符合 OpenClaw SKILL.md frontmatter 规范
- 可被正确加载、发现和维护

## 2. Skill 定义

OpenClaw Skill 是具有标准化 SKILL.md 文件的独立功能单元。
Skill 存储在 `skills/` 目录下，每个 Skill 一个子目录：

```
skills/
└── {skill-name}/
    └── SKILL.md   ← 唯一入口文件
```

## 3. SKILL.md 强制格式

```yaml
---
name: {skill-name}          # 唯一标识，英文、数字、下划线
description: {一句话描述}  # 不超过 160 字节
version: x.y.z              # 语义化版本
metadata:
  openclaw:
    requires: []            # 依赖的其他 skills（可选）
    tags: []               # 标签
---

# {Skill 名称}

## 描述
<!-- 详细描述：做什么、什么时候用、怎么用 -->

## 前置条件
<!-- 使用此 Skill 前必须满足的条件 -->

## 使用方法
<!-- 具体调用方式和参数说明 -->

## 输出
<!-- 执行结果的格式和含义 -->

## 限制
<!-- 使用边界和注意事项 -->

## 示例
<!-- 至少 2 个使用示例 -->
```

## 4. Skill 开发流程

### Phase 1：需求确认

| 步骤 | 操作 | 产出 |
|------|------|------|
| 1.1 | 识别需要 Skill 解决的重复任务 | 需求描述 |
| 1.2 | 确认是否已有相似 Skill（避免重复） | 重用决策 |
| 1.3 | 确定 Skill 名称和功能边界 | 功能范围文档 |
| 1.4 | 分配 Skill 编码（如 KB_SK_XXX） | 编码确认 |

### Phase 2：编写 SKILL.md

| 步骤 | 操作 |
|------|------|
| 2.1 | 创建 `skills/{name}/SKILL.md` |
| 2.2 | 填写 frontmatter（name/description/version 必填） |
| 2.3 | 编写描述、前置条件、使用方法 |
| 2.4 | 编写使用示例（至少 2 个） |
| 2.5 | 列出限制和使用边界 |
| 2.6 | 检查格式：frontmatter 完整、无敏感信息 |

### Phase 3：内部测试

| 步骤 | 操作 | 判定 |
|------|------|------|
| 3.1 | 在目标 Agent 中激活 Skill | 加载成功 |
| 3.2 | 用示例场景执行 Skill | 输出符合预期 |
| 3.3 | 测试边界条件（异常输入） | 失败处理正确 |
| 3.4 | 测试与相关 Skill 的配合 | 无冲突 |

### Phase 4：上线评审

**评审内容：**
- SKILL.md frontmatter 完整（name/description/version 必填）
- 描述准确，无歧义
- 示例可执行
- 无敏感信息泄露
- 命名符合规范（kebab-case）

**评审通过标准：**
- 所有必填字段存在
- 至少 1 个可执行的示例
- 描述与实际行为一致

### Phase 5：发布

| 步骤 | 操作 |
|------|------|
| 5.1 | 将 frontmatter `status` 设为 `active`（如有此字段） |
| 5.2 | 更新 `SOP索引.md`，新增 Skill 条目 |
| 5.3 | 在 `knowledge/cards/` 中创建关联卡片（如需要） |
| 5.4 | 在 `daily/YYYY-MM-DD.md` 中记录上线 |

## 5. Skill 命名规范

| 规则 | 正确 | 错误 |
|------|------|------|
| 英文小写 | `cron-manager` | `CronManager` |
| 字母+数字+下划线 | `daily-report` | `daily report` |
| 体现功能 | `agent-coordinator` | `skill-01` |
| 动宾结构（方法类） | `file-version-control` | `control` |

## 6. Skill 加载优先级

OpenClaw Skill 加载顺序（前者优先）：

1. workspace 内 `skills/`
2. Agent 自身 `agents/{name}/skills/`
3. 全局 `skills/`（如有）
4. 捆绑内置 Skills

## 7. Skill 撤销

如 Skill 不再需要：

1. 将 `skills/{name}/` 目录移至 `memory/archive/skills/`
2. 在目录名加 `_obsolete_{日期}` 后缀
3. 从 `SOP索引.md` 中移除或标注为 obsolete
4. 在 `daily/` 中记录撤销原因
