# Agent 下线流程 SOP

> 编码：AG_SOP_003_v1.0.0 | 版本：1.0.0 | 2026-06-27 | 状态：active
> 监管要求：IATF16949:2016 §7.5 成文信息控制
> 适用范围：Agent 从 OpenClaw 生态撤销的全流程

## 1. 目的

规范 Agent 从 Active 到 Obsolete 的下线流程，确保：
- 职责有接替方案（不出现职能真空）
- 相关文件有归档路径
- 受控文件清单及时更新
- 历史记录保留可追溯

## 2. 下线原因分类

| 原因 | 处理策略 |
|------|---------|
| 架构重组（职能合并） | 职责转移，旧 Agent 下线 |
| 职能撤销（不再需要） | 直接下线，归档 |
| 严重故障（Agent 不可恢复） | 立即冻结，评审后下线 |
| 重复建设（与其他 Agent 重叠） | 合并，保留一个 |

## 3. 下线流程

```
下线评估 ──→ 职责转移 ──→ 文件归档 ──→ 受控清单清理 ──→ 下线确认
```

### Phase 1：下线评估

**评估者：** CEO

| 评估项 | 内容 |
|--------|------|
| 下线原因 | 为什么要下线？ |
| 影响分析 | 下线后哪些职责无人承担？ |
| 接替方案 | 职责转移到哪个 Agent？ |
| 风险评估 | 是否有风险？ |

**决策：**
- 可下线 → 进入 Phase 2
- 暂不下线 → 记录原因，关闭评估
- 需整改 → 制定整改计划

### Phase 2：职责转移

**执行者：** CEO

| 步骤 | 操作 |
|------|------|
| 2.1 | 识别所有待转移职责 |
| 2.2 | 确认接替 Agent |
| 2.3 | 向接替 Agent 同步职责上下文 |
| 2.4 | 更新接替 Agent 的 AGENTS.md（新增职责） |
| 2.5 | 通知相关方（飞书/企业微信） |

**职责转移清单：**
```markdown
## 职责转移清单：{下线Agent}

| 原职责 | 接替Agent | 转移日期 | 状态 |
|--------|-----------|---------|------|
| 职责1 | Agent B | YYYY-MM-DD | 已转移 |
| 职责2 | Agent C | YYYY-MM-DD | 待转移 |
```

### Phase 3：文件归档

**执行者：** CEO

| 文件类型 | 归档操作 |
|---------|---------|
| 工作区文件 | 移至 `memory/archive/agents/{agent-id}_obsolete_{日期}/` |
| Skills | 移至 `memory/archive/agents/{agent-id}_obsolete_{日期}/skills/` |
| Cron 定义 | 撤销 Cron job（cron remove） |
| knowledge/ | 知识库文件评估是否保留 |
| memory/ | 重要记忆卡片转移至 MyBrain |

**归档操作：**
```bash
# 示例
mv ~/.openclaw/agents/vp1 ~/.openclaw/workspace/memory/archive/agents/vp1_obsolete_2026-06-27/
```

### Phase 4：受控清单清理

**执行者：** CEO

| 步骤 | 操作 |
|------|------|
| 4.1 | 从 `knowledge/system/agents.md` 移除 Agent 条目（标注为 obsolete） |
| 4.2 | 从 `system/受控文件清单.md` 移除相关文件 |
| 4.3 | 从 `system/SOP索引.md` 移除 Skill 条目 |
| 4.4 | 确认无其他文件引用该 Agent（更新所有引用） |

### Phase 5：下线确认

**执行者：** CEO

| 步骤 | 操作 |
|------|------|
| 5.1 | 确认所有职责已转移 |
| 5.2 | 确认所有文件已归档 |
| 5.3 | 确认受控清单已更新 |
| 5.4 | 在 `daily/YYYY-MM-DD.md` 记录下线事件 |
| 5.5 | 向相关方通知 Agent 已下线 |

## 4. 紧急下线（快速冻结）

如 Agent 发生严重故障需要立即冻结：

1. **立即停止**：停止所有对该 Agent 的调用
2. **临时接替**：CEO 直接接管或指派临时接替者
3. **快速归档**：关键文件迁移至 archive
4. **补办流程**：完成紧急冻结后，补全 Phase 1-5

## 5. 已下线 Agent 的处理

| 状态 | 说明 | 是否可见 |
|------|------|---------|
| archived | 已归档，正常保存 | 不可见（不参与调度） |
| referenced | 被引用但已下线 | 可见，但标注为 obsolete |

## 6. 禁止事项

- **禁止**在职责未转移前直接删除 Agent
- **禁止**在 Cron job 未清理前归档 Agent
- **禁止**在受控清单未更新前确认下线完成
- **禁止**删除 Agent 后不通知相关方

## 7. 记录要求

| 记录 | 保存位置 | 保存期限 |
|------|---------|---------|
| 下线报告 | `daily/事件报告_YYYY-MM-DD.md` | 永久 |
| 归档目录 | `memory/archive/agents/` | 1 年后评估销毁 |
| 职责转移清单 | 下线报告中 | 永久 |
