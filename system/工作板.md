### 队列模式（Multi-Agent Routing）

```
外部请求 → CEO 主会话（入口队列）
    │
    ├── [P0] 紧急 → CEO 立即处理
    │
    ├── [技术] 代码/架构/QA/硬件/软件/产品
    │       └── sessions_send → agent:cto:main
    │               └── CTO 按 Skill 分发到对应 Agent
    │
    ├── [运营] 项目/质量/销售/研究
    │       └── sessions_send → agent:coo:main
    │               └── COO 按 Skill 分发到对应 Agent
    │
    ├── [战略] 合规/规划/分析
    │       └── sessions_send → agent:ca:main
    │
    └── [定时] Cron jobs
            └── 各域 Agent 独立 session 自运行
                    └── 结果汇报回 CEO 或写入约定位置
```

### 交接协议

| 工具 | 用途 | 等待方式 |
|------|------|---------|
| `sessions_spawn` | 并行派发多个子任务 | `sessions_yield` |
| `sessions_send` | 向指定 Agent 发消息并等待回复 | 同步等待回复 |
| `sessions_spawn(mode="run")` | 派发后台任务，不等待 | 无需汇报 |
| 定时 Job | 自动触发自运行任务 | 结果写入文件 |
