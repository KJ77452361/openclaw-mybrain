# SOP 标准格式

> 编码：KB_SOP_001_v1.0.0 | 版本：1.0.0 | 2026-06-27 | 状态：active
> 监管要求：IATF16949:2016 §7.5 成文信息控制
> 适用范围：MyBrain 内所有 SOP、作业指导书、工作流程文件

## 1. 目的

统一 MyBrain 生态内所有 SOP 的格式规范，确保：
- 所有 SOP 具有一致的结构和可读性
- 每份 SOP 可独立执行，无需外部参考
- 符合 IATF16949 §7.5 对成文信息的要求

## 2. 适用范围

所有以 SOP /WI / 工作流 命名的文档均须遵循本格式。

## 3. 强制结构

每份 SOP 必须按顺序包含以下章节：

### 3.1 文件头（frontmatter）

```yaml
---
title: {SOP标题}
type: SOP | WI | workflow
code: {编码}_{序号}_v{major}.{minor}.{patch}
version: {major}.{minor}.{patch}
created: {YYYY-MM-DD}
updated: {YYYY-MM-DD}
tags: [{标签}]
regulatory: IATF16949:2016 §7.5
owner: {负责人}
status: active | draft | obsolete
---
```

### 3.2 封面信息块

```markdown
> 编码：{code} | 版本：{x.y.z} | {YYYY-MM-DD} | 状态：{status}
> 监管要求：IATF16949:2016 §7.5（如适用）
> 适用范围：{范围描述}
> 负责人：{owner}
```

### 3.3 必有章节

| 章节 | 要求 | 说明 |
|------|------|------|
| 1. 目的 | 强制 | 为什么要做这件事 |
| 2. 适用范围 | 强制 | 谁/什么场景适用 |
| 3. 职责 | 强制 | 各角色的职责定义 |
| 4. 术语/定义 | 建议 | 相关术语解释 |
| 5. 流程/操作步骤 | 强制 | 逐步操作指令 |
| 6. 判定标准 | 建议 | 通过/失败的判定条件 |
| 7. 记录要求 | 强制 | 执行后保留什么记录 |
| 8. 相关文件 | 建议 | 关联文档 |
| 9. 变更记录 | 强制 | 版本历史 |

### 3.4 版本 Footer

```markdown
---

*version x.y.z · King · YYYY-MM-DD*
```

## 4. SOP 分类

| 类型 | 后缀 | 编码前缀 | 说明 |
|------|------|---------|------|
| 作业程序 | SOP | KB_SOP_* | 跨域、复杂流程 |
| 工作指导 | WI | KB_WI_* | 单一步骤指导 |
| 工作流 | workflow | KB_WF_* | 多角色协作路径 |
| 检查表 | checklist | KB_CL_* | 清单确认类 |

## 5. 编号规则

```
{域代码}_{类型代码}_{序号}_v{版本}
```

| 域代码 | 适用范围 |
|--------|---------|
| KB | 知识库通用 |
| AG | Agent 系统 |
| WM | Workspace 管理 |
| QL | 质量/合规 |

序号由知识库管理员统一分配，防止重号。

## 6. 版本更新规则

| 变更级别 | 触发条件 | 版本变化 |
|---------|---------|---------|
| patch | 错别字、格式修正 | v{x}.{y}.{z} → v{x}.{y}.{z+1} |
| minor | 内容增减 <20% | v{x}.{y}.{z} → v{x}.{y+1}.{0} |
| major | 重写 ≥20% | v{x}.{y}.{z} → v{x+1}.{0}.{0} |

## 7. 禁止事项

- **禁止**在 SOP 正文中使用"一般为""大概""可能"等模糊表述
- **禁止**省略"职责"章节（导致责任不清）
- **禁止**在 SOP 中嵌入真实密钥或凭证
- **禁止**创建无法独立执行的 SOP（必须有步骤、有判定）

## 8. SOP 审核清单

新建或修订 SOP 时，逐项确认：

- [ ] frontmatter 必填字段完整（title/type/code/version/owner/status）
- [ ] 有封面信息块（编码/版本/状态/监管要求）
- [ ] 包含 1 目的 / 3 职责 / 5 操作步骤 / 7 记录要求 / 9 变更记录
- [ ] 操作步骤为可执行指令（非描述性文字）
- [ ] 判定标准明确（可判断成功/失败）
- [ ] 无模糊表述
- [ ] 无敏感信息泄露
- [ ] 版本 footer 正确
