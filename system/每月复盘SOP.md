# 月/季度复盘 SOP

> 编码：KB_SOP_011_v1.0.0 | 版本：1.0.0 | 2026-06-27 | 状态：active
> 监管要求：IATF16949:2016 §7.5 成文信息控制
> 适用范围：CEO 的月度（每月末）和季度（每季度末）深度复盘

## 1. 目的

以月/季度为周期进行深度战略复盘，确保：
- 全局视角审视 OpenClaw 生态健康度
- 识别需要结构性改变的系统性问题
- 评估自进化系统的实际效果
- 制定下月/季度的战略方向

## 2. 触发时机

| 类型 | 时间 | 执行者 | 输出 |
|------|------|--------|------|
| 月度复盘 | 每月最后一天 20:00 | CEO | `daily/月报_YYYY-MM.md` |
| 季度复盘 | 每季度末天 20:00 | CEO | `daily/季报_YYYY-QN.md` |

## 3. 月度复盘流程

```
数据聚合（4份周报） → 六维度分析 → 战略评估 → 下月计划 → 自进化调整
```

### Phase 1：数据聚合

**收集以下数据：**
- 本月 4-5 份周报（`daily/周报_YYYY-WXX.md`）
- 本月所有日记录（`daily/YYYY-MM-DD.md`）
- 本月问题单汇总（`inbox/`）
- 本月受控文件变更（`system/受控文件清单.md`）
- 本月 Cron 执行数据

### Phase 2：六维度分析

| 维度 | 问题 | 产出 |
|------|------|------|
| **目标达成** | 本月原定 KPI 完成情况？ | KPI 完成率 |
| **生态健康** | 各 Agent 运行是否正常？ | 健康度评分 |
| **知识积累** | 本月新增了多少高质量内容？ | 知识积累量 |
| **质量趋势** | 敏感信息泄露、错误率趋势？ | 质量指标 |
| **自进化效果** | 本月触发了多少次自进化？效果？ | 进化效果评分 |
| **风险扫描** | 有哪些新的系统性风险？ | 风险清单 |

### Phase 3：战略评估

使用 SWOT 框架：

| 维度 | 内容 |
|------|------|
| **优势 S** | 什么做得好，值得保持？ |
| **劣势 W** | 什么做得不好，需要改进？ |
| **机会 O** | 有什么新机会值得探索？ |
| **威胁 T** | 有什么风险需要防范？ |

### Phase 4：下月计划

| 类型 | 内容 |
|------|------|
| 核心目标 | 1-3 个战略级目标 |
| 关键行动 | 5-10 个关键行动 |
| 资源需求 | 需要什么资源（时间/人力/工具） |
| 成功指标 | 如何衡量下月是否成功 |

## 4. 季度复盘（扩展月度流程）

季度复盘在月度基础上增加：

### 额外分析维度

| 维度 | 内容 |
|------|------|
| **生态演进** | 相比上季度，生态有哪些实质性进步？ |
| **SOP 有效性** | 现有 SOP 是否仍然适用？需要修订哪些？ |
| **自进化闭环评估** | 自进化三层架构是否有效运行？ |
| **架构评估** | CTO 域架构是否需要调整？ |
| **战略方向** | 下季度战略方向是否需要调整？ |

### 季度审查清单

| 检查项 | 标准 | 判定 |
|--------|------|------|
| 受控文件合规率 | ≥95% | ✅/⚠️/❌ |
| frontmatter 合规率 | 100% | ✅/⚠️/❌ |
| SOP 执行率 | 有执行记录 | ✅/⚠️/❌ |
| 自进化触发次数 | ≥1 次/月 | ✅/⚠️/❌ |
| 知识积累增长 | 正增长 | ✅/⚠️/❌ |
| 敏感信息泄露 | 0 次 | ✅/❌ |

## 5. 输出规范

| 输出 | 保存位置 | 用途 |
|------|---------|------|
| 月度复盘报告 | `daily/月报_YYYY-MM.md` | 永久战略记录 |
| 季度复盘报告 | `daily/季报_YYYY-QN.md` | 永久战略记录 |
| SOP 修订需求 | 触发 KB_SOP_003 | 流程优化 |
