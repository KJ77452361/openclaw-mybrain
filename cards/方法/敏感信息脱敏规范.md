# 敏感信息脱敏规范（方法卡）

> 编码：KB_CAR_方法_002_v1.0.0 | 版本：1.0.0 | 2026-06-27 | 状态：active
> 说明：快速参考卡，详细规范见 `system/敏感信息脱敏规范.md`（QL_SOP_002_v1.0.0）

## 摘要

识别和处理 OpenClaw 生态内的敏感信息，防止凭证泄露。

## 敏感信息类型与脱敏规则

| 类型 | 识别特征 | 脱敏规则 | 示例 |
|------|---------|---------|------|
| API Key | `sk-` 开头 | `sk-***` | `sk-abc123...` → `sk-***` |
| App Secret | 任何位置 | `***` | `app_secret=xxx` → `***` |
| Bearer Token | `Bearer ` 开头 | `Bearer ***` | `Bearer eyJhbG...` → `Bearer ***` |
| 用户密码 | 任何位置 | 禁止明文 | `password=xxx` → 删除 |
| 私钥/SSH Key | `-----BEGIN` 开头 | 禁止明文 | `-----BEGIN RSA...` → 删除 |
| 身份证号 | 18位数字 | `***/**/****1234` | `310101199001011234` |
| 手机号 | 11位数字 | `138****5678` | `13812345678` → `138****5678` |
| 邮箱 | 含@ | `a***@b.com` | `admin@example.com` → `a***@b.com` |

## 强制扫描时机

| 时机 | 操作 |
|------|------|
| 文件发布前 | 手动 grep 检查 |
| 每周一 Cron | 自动化全量扫描（见 QL_SOP_002 §5） |
| 收到泄露告警 | 立即全量扫描 |

## 泄露处理7步法

1. 确认泄露范围（grep 全量）
2. 立即 Revoke 受影响凭证
3. 替换为新凭证（如需要）
4. 通知相关方（如涉及外部服务）
5. 清除所有历史明文记录
6. 审查为何未检测到（流程漏洞）
7. 更新防护措施（QL_SOP_002 §6.4）
