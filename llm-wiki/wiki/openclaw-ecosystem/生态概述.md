# OpenClaw 生态概述

## 核心定位

OpenClaw 是一个**多 Agent 自进化系统**，通过 LLMs + 工具链 + 记忆系统，实现个人/组织的第二大脑。

OpenClaw 的核心理念与 Karpathy LLM Wiki 一脉相承：
> **知识编译一次，常驻最新，持续自进化。**

---

## 架构分层

| 层级 | 说明 |
|------|------|
| **对外入口** | CEO（智能体助手）作为用户唯一入口 |
| **决策中枢** | CA（战略分析）+ COO（运营协调）+ CTO（技术架构）|
| **业务执行** | 各域 Agent（pm/qm/rs/sa/he/se/pe/architect 等）|
| **知识沉淀** | MyBrain 知识库 + LLM Wiki |
| **基础设施** | OpenClaw Gateway + Cron + Session |

---

## 域（Domain）结构

### CEO 域
- **CEO**：唯一出口、决策中枢、记忆引擎
  - 5个 Skills：agent-coordinator、cron-manager、knowledge-manager、memory-manager、standard-auditor
  - 知识库：`~/.openclaw/agents/ceo/`

### CTO 域（技术）
| Agent | 角色 | Skill |
|-------|------|-------|
| architect | 架构设计与评审 | architecture-review |
| fullstack_engineer | 代码实现检查 | code-review |
| qa_engineer | 质量审计与测试 | quality-audit |
| he | 硬件运维 | hardware-ops |
| se | 软件设计 | software-design |
| pe | 产品需求 | product-requirements |

### COO 域（运营）
| Agent | 角色 | Skill |
|-------|------|-------|
| pm | 项目管理 | project-management |
| qm | 质量指标 | quality-metrics |
| rs | 研究调研 | — |
| sa | 销售分析 | — |

### CA 域（合规/战略）
- **ca**：战略分析与规划（合规总监）

---

## 知识流

```
用户输入 → CEO（决策/分发）
    ↓
CA（战略分析）→ COO（任务编排）→ CTO（技术评审）→ 各执行Agent
    ↓                                    ↓
记忆系统（MEMORY.md / daily/）←——→ MyBrain 知识库
                                          ↓
                                   LLM Wiki（持久积累）
```
